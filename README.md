# Discord-бот экономики «Серебро» 🪙

Бот с системой экономики: баланс, переводы, лидерборд, начисление/списание администраторами.
Каждая команда работает и через префикс (`!cmd`), и через слэш (`/cmd`).

## Команды

| Команда | Слэш | Кто | Что делает |
|---|---|---|---|
| `!help` | `/help` | все | список доступных **именно тебе** команд (зависит от роли) |
| `!bal [@кто]` | `/bal` | все (в канале экономики) | показать баланс |
| `!transfer @кому сумма` | `/transfer` | все (в канале экономики) | передать серебро |
| `!lb` | `/lb` | все (в канале экономики) | таблица лидеров, по 10 на странице, листается кнопками |
| `!give-balance @кому сумма` / `!gb` | `/give-balance` | только админ | начислить серебро |
| `!remove-balance @кому сумма` / `!rb` | `/remove-balance` | только админ | списать серебро |

- **Не-администраторы** могут вызывать `!bal`, `!transfer`, `!lb` только в канале, указанном в `ECONOMY_CHANNEL_ID`.
- **Администраторы** (право «Администратор» на сервере) могут вызывать команды в любом канале.

## Приветствие новичков

Когда новый участник заходит на сервер, бот пишет ему **в личку** (на английском)
сообщение со списком каналов, которые стоит посмотреть. Если у новичка закрыты ЛС —
бот дублирует приветствие в резервный канал `WELCOME_CHANNEL_ID` с упоминанием.

Каналы для подсказки задаются в `.env` через `WELCOME_CHANNELS` в формате `Метка:ID`
через запятую, например: `WELCOME_CHANNELS=Rules:123..., Introductions:456...`.
Метку можно опустить — тогда будет просто ссылка на канал. Пусто — секция не показывается.
Нужен включённый **SERVER MEMBERS INTENT** (он уже требуется для команд).

**Текст приветствия** редактируется в файле `welcome.json` (рядом с `bot.py`):
- `title`, `intro`, `footer` — заголовок, вступление, подпись. Плейсхолдеры: `{guild}`
  (имя сервера), `{member}` (упоминание новичка), `{count}` (число участников).
- `channel_line` / `channel_line_no_label` — как выглядит строка канала (плейсхолдеры `{label}`, `{id}`).
- `no_channels` — текст, если `WELCOME_CHANNELS` пуст.
- `color` — цвет полоски (`0xRRGGBB`), `show_guild_icon` — показывать ли иконку сервера.

JSON перечитывается при каждом входе нового участника — **перезапуск бота для смены
текста не нужен**, достаточно сохранить файл.

## Игровые роли

Игроки выбирают себе игровые роли (чисто внутриигровые, **не** Discord-роли — хранятся в БД),
а администраторы проставляют каждой роли приоритет (1–3).

**Игрок** (в канале с панелью, ставится командой `/setup-game-panel`):
- **📝 Выбрать роли** — открывается постраничный выбор: каталог любого размера, роль
  отмечается кнопкой ✅, счётчик «Выбрано: X/10», листание ◀▶. Максимум 10 ролей.
  «Готово» — заявка уходит администраторам.
- **📋 Мои роли** — показывает выбранные роли, приоритеты и статус (на рассмотрении / утверждено).
- Чтобы изменить роли — снова «Выбрать роли» (создаётся новая заявка, приоритеты сбрасываются).

**Администраторы** (в канале `ADMIN_CHANNEL_ID`):
- При новой заявке бот публикует панель: на каждую роль ряд `[Роль] [1] [2] [3] [✖]`
  (текущий приоритет подсвечен зелёным), ниже «➕ Добавить роль» и «✅ Утвердить».
- Можно убрать роль (✖), добавить новые (➕, тот же постраничный выбор), задать приоритет.
- «✅ Утвердить» — заявка становится активной, игроку приходит уведомление в ЛС.
- `/review @игрок` — открыть панель игрока для редактирования (задать/убрать роли и приоритеты).
  Работает и для игрока, который сам заявку **не подавал** — откроется пустая панель,
  роли назначаются кнопкой `➕ Add Role`.
- `/roster` — список всех игроков с их ролями и приоритетами (по страницам, ephemeral).

**Каталог ролей** — файл `roles.json` (рядом с `bot.py`), перечитывается на лету:
```json
[
  { "key": "sniper", "name": "Sniper", "emoji": "🎯", "description": "..." }
]
```
`key` — уникальный идентификатор (латиницей), `name` — отображаемое имя, `emoji` и
`description` необязательны. Ролей может быть сколько угодно (в одном меню выбора Discord
показывает до 20 на странице, листается кнопками).

## Сборка состава (compositions)

Распределение отметившихся игроков по ролям набора с учётом приоритетов.

**Наборы** задаются в `sets/<имя>.json` — тиры по 5: первые 20 ролей, потом +5 (до 25),
потом +5 (до 30) и т.д. Роли можно писать ключом (`bms`) или именем (`BMS`) из `roles.json`:
```json
{ "name": "main", "tiers": [ ["BMS","BMS", "...20 ролей"], ["...5..."], ["...5..."] ] }
```

**Команды** (только админ):
- `/sets` — список наборов и какие уже собраны.
- `/compose <набор> <ссылка-или-id-сообщения>` — парсит всех, кто поставил реакцию на
  сообщение, и распределяет по ролям. Открывает панель правки. Если состав уже собран —
  спросит подтверждение (пересборка с нуля стирает ручные правки).
- `/composition <набор>` — заново открыть панель правки.
- `/publish <набор> [канал]` — опубликовать готовый состав игрокам.
- `/vc-check <набор>` — берёт голосовой канал, в котором сейчас ты, и пингует тех
  назначенных в составе игроков, кого в этом голосовом канале нет.

**Алгоритм:** тир за тиром (сначала полностью 20, потом +5…); внутри тира — по приоритету
1→2→3 (затем без приоритета); на каждом уровне максимальное паросочетание (занять как можно
больше слотов), дефицитные игроки (у кого меньше доступных ролей набора) идут вперёд, среди
равных — случайно. Один игрок — одна роль. Кому места не нашлось — в списке «Unassigned»
с их ролями, дальше правишь руками в панели (выбрать слот → назначить/снять игрока).

## Настройка

### 1. Создать приложение бота
1. Зайдите на https://discord.com/developers/applications → **New Application**.
2. Вкладка **Bot** → **Reset Token** → скопируйте токен.
3. На той же вкладке включите **Privileged Gateway Intents**:
   - ✅ **MESSAGE CONTENT INTENT**
   - ✅ **SERVER MEMBERS INTENT**
4. Вкладка **OAuth2 → URL Generator**: отметьте scopes `bot` и `applications.commands`,
   из прав бота достаточно `Send Messages`, `Embed Links`, `Read Message History`.
   Откройте сгенерированную ссылку и пригласите бота на сервер.

### 2. Конфигурация
```powershell
copy .env.example .env
```
Откройте `.env` и заполните:
- `DISCORD_TOKEN` — токен из шага 1.
- `ECONOMY_CHANNEL_ID` — ID канала для команд не-админов (Режим разработчика → ПКМ по каналу → Копировать ID).
- `GUILD_ID` — ID вашего сервера (необязательно, но слэш-команды появятся мгновенно).

### 3. Запуск
```powershell
.\.venv\Scripts\python.exe bot.py
```
(venv уже создан. Если нужно заново: `python -m venv .venv` → `.\.venv\Scripts\python.exe -m pip install -r requirements.txt`.)

## Файлы
- `bot.py` — запуск, интенты, синхронизация слэш-команд.
- `cogs/economy.py` — все команды, проверки прав/канала, пагинация лидерборда.
- `database.py` — таблица `balances` (SQLite), баланс/перевод/лидерборд.
- `config.py` — чтение `.env`, оформление валюты.
